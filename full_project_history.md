# 📜 บันทึกประวัติการติดตั้งระบบ Motorcycle Shop (ฉบับสมบูรณ์)

บันทึกนี้รวบรวมเหตุการณ์และทางเทคนิคทั้งหมดที่เกิดขึ้นระหว่างการ Deploy ระบบขึ้น Cloud (Render.com)

---

## 🏗️ 1. ช่วงการเตรียมโครงสร้าง (System Architecture)
- **แพลตฟอร์ม**: เลือกใช้ **Render.com** (เนื่องจากใช้งานง่ายและมีฐานข้อมูลในตัว)
- **เทคโนโลยี**: Docker Container (ใช้ไฟล์ `Dockerfile`)
- **ฐานข้อมูล**: PostgreSQL (Managed by Render)

### การตั้งค่าตัวแปร (Environment Variables)
- `DATABASE_URL`: ใช้รูปแบบ `postgres://user:password@host/db`
- `ASPNETCORE_ENVIRONMENT`: ตั้งเป็น `Production`
- `PORT`: Render จะกำหนดให้เอง และโปรแกรมถูกแก้ให้รับค่าได้อัตโนมัติ

---

## 🛠️ 2. ช่วงการแก้ไขปัญหาเฉพาะหน้า (Troubleshooting)

### ปัญหาที่ 1: การเชื่อมต่อฐานข้อมูล
- **ปัญหา**: เชื่อมต่อ Neon/Railway ไม่สำเร็จเนื่องจากรหัสผ่านมีอักขระพิเศษ
- **วิธีแก้**: เขียนโค้ดใน `Program.cs` ใหม่ให้ใช้ `Uri.UnescapeDataString` เพื่อถอดรหัสรหัสผ่านให้ถูกต้องสากล

### ปัญหาที่ 2: การย้ายข้อมูล (Data Migration)
- **ปัญหาครั้งแรก**: ไฟล์สำรองข้อมูล (`motorshop_backup.sql`) มีขนาดว่างเปล่า
- **การแก้ไข**: ตามหาตัวโปรแกรม `pg_dump.exe` ในเครื่องจนเจอที่ `C:\Program Files\PostgreSQL\18\bin\` และรันการ Backup ของจริงออกมา
- **ปัญหาเรื่องภาษา**: ไฟล์ SQL จาก PowerShell เป็นแบบ UTF-16 ซึ่ง Cloud อ่านไม่ได้
- **วิธีแก้**: แปลงไฟล์เป็น UTF-8 (No BOM) ในชื่อ `motorshop_final.sql` ก่อนส่งขึ้นเว็บ

---

## 🔐 3. ช่วงการจัดการความปลอดภัย (Authentication Fix)

### ปัญหาที่ 3: ล็อกอินไม่ได้ (Password Mismatch)
- **สาเหตุ**: โค้ดที่ใช้สร้างผู้ใช้ใหม่ (UserController) กับโค้ดที่ใช้ล็อกอิน (AuthService) ใช้วิธีแปลงรหัสผ่านไม่เหมือนกัน (ฝั่งหนึ่งใช้ Hex อีกฝั่งใช้ Base64+Salt)
- **วิธีแก้**: 
    1. ปรับปรุง `UserController` ให้ใช้ Salt `"MotorcycleShop2024"` และ Base64 เหมือนกัน
    2. แก้ไข `AuthService.VerifyPassword` ให้ฉลาดขึ้น โดยสามารถอ่านรหัสผ่านได้ทั้งแบบเก่า (Hex) และแบบใหม่ (Base64) เพื่อความยืดหยุ่น

### ปัญหาที่ 4: ตัวพิมพ์เล็ก-พิมพ์ใหญ่ (Case Sensitivity)
- **วิธีแก้**: ปรับการค้นชื่อผู้ใช้ในฐานข้อมูลให้เป็น `ToLower()` (ตัวพิมพ์เล็กทั้งหมด) ทำให้พิมพ์ `Admin` หรือ `admin` ก็เข้าได้เหมือนกัน

---

## 🚨 4. ระบบกู้ภัย (Rescue Mode)
- **Admin Rescue**: มีการฝังโค้ดใน `Program.cs` ว่าเมื่อใดก็ตามที่เว็บเริ่มทำงาน จะบังคับให้ชื่อ `admin` มีรหัสผ่านเป็น `password` เสมอ เพื่อป้องกันกรณีลืมรหัสผ่านหรือฐานข้อมูลมีปัญหา

---

## 💻 5. สรุปคำสั่งสำคัญที่เคยใช้
- **รัน Backup**: `& "C:\Program Files\PostgreSQL\18\bin\pg_dump.exe" -U postgres -d postgres -f "motorshop_real_data.sql"`
- **ส่งข้อมูลขึ้น Cloud**: (ใช้คำสั่ง `psql` พร้อมรหัส External URL ของ Render)
